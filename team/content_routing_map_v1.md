# Content Routing Map v1

This file maps existing project materials to the launch team structure. Use it as the rule for where future work should be added.

## Recommended Folder Logic

Keep the current `brand-playbook/` as the master brand library. It is like the project's reference book.

Use the team functions to decide ownership:

- Founder / Brand Owner owns final decisions.
- Product / Merchandising / Sourcing owns SKU, inventory, suppliers, cost, and replenishment.
- Creative / Content / Editorial owns visual direction and content assets.
- Growth / Marketing owns SEO, creator, affiliate, social, paid tests, and traffic.
- E-commerce / Website / Conversion owns Shopify, product pages, quiz, checkout, and conversion.
- Operations / Fulfillment / Customer Experience owns shipping, packaging, returns, customer questions, and service process.
- Finance / Analytics owns model, cash flow, SKU profitability, and scenario planning.
- Admin / Legal / People owns file organization, permissions, contracts, and business admin.

## What Not To Move Yet

Do not move everything immediately into department folders. Many files are cross-functional and useful in more than one place.

Better rule:

- Keep `brand-playbook/` as the master reference library.
- Put new working files into the department/function folder when they are created.
- Use this routing map to decide who owns each file.
